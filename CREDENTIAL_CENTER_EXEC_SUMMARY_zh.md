# 凭据中心改造一页摘要（汇报版）

## 一、项目目标

围绕“Agent 访问远程资源凭据治理”完成改造，目标是：

- 支持 **Agent 独立凭据**（按 Agent ID 隔离）。
- 支持 **全局共享凭据**（可跨 Agent 引用）。
- 在 Console 提供凭据 **可视化 CRUD 管理**。
- 打通模型（Provider）、MCP、Dify 的凭据引用与运行时注入主调用链（部分路径按上下文条件生效）。
- 分阶段补齐安全：先功能闭环，再加密与审计。

## 二、交付范围（已完成）

### 1) 功能一期

- 凭据模型与作用域：`agent/global`
- 凭据 CRUD API 与列表查询
- 凭据引用对象（`credential_ref` + `field_map`）
- Provider/MCP/Dify 调用链接入凭据解析
- 旧配置迁移为凭据引用
- Console 新增 Credentials 页面与引用入口

### 2) 安全二期（第一批）

- 凭据敏感字段接入 `secret_store` 加密落盘
- 明文查看能力接入审计日志（可查询）

## 三、关键结果

- 形成端到端闭环（以主调用链为主）：  
  **配置 -> 存储 -> 引用 -> 注入 -> 调用 -> 审计**
- 支持“Agent 独立 + 全局共享”的目标能力
- 前端已补充“目标 Agent”选择，确保按 Agent 身份显式管理凭据
- 联调脚本已沉淀，可重复回归并自动清理测试数据

## 四、测试与联调结论

- 测试口径说明：静态核对所列 4 个 pytest 文件共 `17` 个用例（文档中的 `17 passed` 指向该集合）
- 覆盖项包括：
  - 凭据存储/解析单测
  - 审计日志单测
  - MCP 与运行配置集成测试
  - 敏感链路联调（明文查看审计、MCP 迁移、Dify invoke 受控异常）
- 结果：核心功能与关键安全能力可用（最终通过状态以本地/CI 实际运行日志为准）

## 五、主要问题与修复

- `pytest` 缺失：已安装并恢复测试链路
- 审计日志目录不存在：已在写入前自动建目录
- 后端循环依赖：已通过 `TYPE_CHECKING` 导入解除
- 前端构建类型错误：已修复 `credential_ref` 的 `null` 类型兼容
- 页面不显示 Credentials：已确认旧静态包导致并重建前端

## 六、当前状态判断

**可进入试运行/灰度阶段。**  
当前已具备可运营的凭据中心能力，满足“按 Agent 独立配置凭据并可在模型/MCP/Dify 主调用链中引用”的核心业务诉求。  
注：现阶段注入口径为“主调用链已接入，部分路径按上下文条件生效”，并非所有分支无条件注入。
